1. tag 是分類維度
   - `tw`: 有機會看到臺灣供應商或產業鏈相關資料
   - `free`: 免費可用
   - `paid`: 需要付費或訂閱
   - `nodata`: 只有入口網站，沒有實際資料，直接 drop
   - `dirty`: 資料雜亂、需要清洗
   - `clean`: 資料結構化、可直接使用

2. 資料來源有兩個難度層級
   - `簡單`: 資料乾淨且集中，通常是 API / CSV，一個來源可拿到多家公司資料 不需使用爬蟲 
   - `中等`: 沒有提供API或者不提供下載但是這個資料來源蒐集了多個公司資料 可用爬蟲取得. 資料集中 能夠透過爬蟲取得. 
   - `困難`: 若資料分散到需要一來源一爬蟲，維護成本過高，邊際效益低，直接 `drop`

3. 找 URL 的原則很明確
   - 不是只找到官方網站就算完成
   - 必須找到明確的資料來源
   - 在深入找之前，要先驗證這個網站真的有你要的資料
   - 不要假設資料存在，然後無止盡往下挖

4. 付費 和 被阻擋 要分開標記
   - `付費`: 用錢可解決
   - `阻擋`: 要靠更高階爬蟲，例如 Playwright 類型的方式處理
