  需求分組

  1. 資料來源

  - 你在評估低成本、可 API、自動化、接近 Bloomberg
    Terminal 的替代資料入口。

  - 你想比較 OpenBB、EODHD、Finnhub、Twelve Data、
    FMP、FRED、ETF issuer data、Google Trends、
    Reddit、OpenSecrets、USAspending.gov 等來源，重點
    不是名氣，而是穩定、便宜、可量化、可批次處理。

  - 你明確關心「美國資料是否比台灣更適合投資研究」，
    並想確認台灣是資料真的不足，還是只是你沒找到。

  - 你想用美國端資料反推台灣供應鏈：如果台灣公司供應
    鏈資料不完整，就從美國大公司、設備商、終端需求端
    查台灣供應商。

  - 你想找接近 BOM / 零件層級 / 供應鏈圖譜的資料來
    源，例如 TechInsights、TrendForce、SemiAnalysis、
    Omdia、Counterpoint、Gartner、IDC、teardown
    reports、Panjiva、ImportGenius、Datamyne、US
    import manifest、ImportYeti。

  - 你不只看取得成本，也看處理成本：資料是否集中、是
    否結構化、是否能 API/CSV/JSON、是否需要 LLM、是否
    要人工逐家公司下載。

  - 你的偏好是：個人可負擔、資訊密度高、資料集中、低
    維護；可以接受資訊少一點，但不能接受高人工成本。

  2. 方式、方法、方案、分析、判斷、預測

  - 你不是單純要資料清單，而是要把資料來源和分析方法
    配對：哪些方法先吃哪些來源，哪些資料適合做哪類分
    析。

  - 你想確認前面「10 種方法被壓成 4 類」是整理整合，
    不是漏掉。

  - 你希望方法要經過資源過濾：人力少、硬體有限，不能
    選重工程、重 GPU、重調參的方法。

  - 你偏好高效率統計 / 數學 / 機器學習方法，例如
    XGBoost、Bayesian、npy/NumPy 類高效資料處理，而不
    是盲目追大型深度模型。

  - 你希望參考 2026 年 5 月附近更前衛但仍務實的方法，
    前提是能在你的硬體與人力限制下落地。

  - 你也在評估 GitHub 開源財經工具或 library，目標是
    把資料整合、特徵工程、驗證、回測流程推進，而不只
    是換一個模型名稱。

  - 你要的是能交給別人執行的規劃文件，但前提是先討
    論、過濾、取得你同意後才寫，不要太早產出定稿。

  真正收斂後的核心需求

  你要建立一套「低成本、可自動化、資料集中、個人可維
  護」的投資研究資料與方法框架。資料面先偏美國，尤其
  用美國公開資料與供應鏈資料反推台灣供應商；方法面則
  偏高邊際效益、低硬體負擔、可驗證的統計/ML 分析流
  程。
