# 前半段

10 種方法，先用「運作機制 + 你實際怎麼用在交易資料」來拆。

## 1. Regime Detection（市場狀態切換）

代表模型：Hidden Markov Model

實際在做什麼：

把市場切成幾種隱藏狀態，例如趨勢、盤整、高波動。  
模型看的不是指標本身，而是報酬分布與波動特性。

怎麼用：

先把歷史資料分段。  
模型標記每段屬於哪種狀態。  
你只在某種狀態下交易。

本質：先分類環境，再決定策略。

## 2. 時間序列分群（Time Series Clustering）

核心工具：Dynamic Time Warping

實際在做什麼：

拿一段價格走勢去找歷史中長得像的段落。  
重點不是數值一樣，而是形狀相似。

怎麼用：

切固定長度窗口。  
比對歷史，找最像的幾段。  
看那些段後面是漲還是跌。

本質：pattern matching，但不是用指標。

## 3. Shape Pattern Mining（形狀挖掘）

代表方法：Matrix Profile

實際在做什麼：

在整段歷史資料中，自動找出反覆出現的波形。

怎麼用：

不先定義 pattern。  
讓系統直接找常出現的形狀。  
再統計這些形狀後的報酬。

本質：讓資料自己告訴你 pattern。

## 4. 自動特徵學習（Feature Learning）

工具：Autoencoder、CNN

實際在做什麼：

把 K 線或報酬序列丟進模型，模型自己壓縮出重要特徵。  
你不必手工定義 RSI、MACD。

怎麼用：

輸入價格序列。  
輸出壓縮特徵，再拿去做預測或分類。

本質：讓模型自己發明指標。

## 5. 樹模型（Tree-based Models）

代表：Random Forest、XGBoost

實際在做什麼：

把很多條件組合起來，自動找到規則。

怎麼用：

輸入你設計的特徵。  
輸出上漲或下跌機率。

本質：非線性規則引擎。

## 6. 序列模型（Sequence Model）

代表：Long Short-Term Memory

實際在做什麼：

讓模型記住前面發生了什麼，再判斷現在。

怎麼用：

輸入一串連續價格。  
輸出下一步的機率或方向。

限制：

容易過度擬合，資料量要求高。

本質：學時間關係。

## 7. 轉折點偵測（Change Point Detection）

實際在做什麼：

找結構突然改變的地方，例如趨勢開始或結束。

怎麼用：

在價格序列中標出轉折點。  
每兩個轉折點之間就是一個波段。

本質：把連續資料切成有意義的段落。

## 8. 機率預測（Probabilistic Forecast）

實際在做什麼：

不是預測會不會漲，而是預測漲 1%、跌 2% 這類機率分布。

怎麼用：

模型輸出分布，不是單一值。  
用來決定倉位，不是只做進出場。

本質：從方向預測轉成風險分布。

## 9. 強化學習（Reinforcement Learning）

實際在做什麼：

模型像交易員一樣做決策，看結果後自己調整。

怎麼用：

建立模擬環境。  
訓練模型最大化報酬。

問題：

很容易學到錯的規則，且很吃資料品質。

本質：直接學策略，不是學預測。

## 10. 橫截面模式（Cross-sectional）

實際在做什麼：

同時看多個商品，不只看單一序列。

怎麼用：

把多個商品一起當輸入。  
模型學相對關係。

本質：市場不是單一序列，而是網絡。

## 核心統一理解

這 10 種方法可以壓成三種邏輯：

- 切結構 -> Regime / Change Point
- 找相似 -> DTW / clustering / pattern mining
- 做預測 -> Tree / LSTM / 機率模型 / RL

## 最實際可落地的一條路

- Change Point -> 切波段
- DTW / clustering -> 找類似波段
- XGBoost -> 預測結果
