# 方法方向

## 核心判準
- 不是看方法名氣，是看邊際效益。
- 邊際效益 = 預期產出 - 取得成本 - 計算成本 - 維護成本 - 資訊成本。
- 只要明顯依賴企業內部資料、深度內餡、或高度即時的微結構資料，就直接排除。
- 只要需求超出 `CPU / 記憶體 / RTX 4050 6G` 的穩定可承受範圍，就直接排除。
- 目標是可重複、可維護、可擴充，不是追最前衛名詞。

## Tier 1: 主線

### 1. Change Point / Regime Detection
- 用途：先切波段，再做後續分析。
- 優點：CPU 可做，成本低，能把連續噪音變成可學的段落。
- 適合：市場狀態切換、趨勢轉換、波動變化。
- 結論：這是最值得先做的入口。

### 2. Tree-based + Cross-sectional
- 用途：把多商品、多特徵做排序與預測。
- 優點：對硬體要求低，訓練和調參成本可控，邊際效益高。
- 適合：相對強弱、因子排名、橫截面預測。
- 結論：這應該是主模型方向，不要先跳深度模型。

### 3. Matrix Profile / Pattern Search
- 用途：找歷史中相似波段與重複型態。
- 優點：比手工找形狀更省力，適合做前兆探索。
- 限制：資料太碎、太分散就不划算。
- 結論：適合當輔助偵測，不適合當唯一主線。

### 4. Probabilistic / Bayesian Methods
- 用途：預測不是單點，而是機率分布或狀態切換。
- 優點：模型通常比重型深度學習輕，解釋性也比較好。
- 適合：風險控制、狀態判斷、不確定性建模。
- 結論：可以放在主線旁邊，不要做成重工程系統。

## Tier 2: 可做但要收斂

### 5. Feature Learning
- 用途：把序列壓成可用特徵。
- 可接受形式：小型 autoencoder、簡單 embedding、tsfresh 類型特徵抽取。
- 不建議：大模型端到端吃原始序列。
- 結論：只能服務主線，不要獨立成黑盒。

### 6. Backtest / Simulation
- 用途：驗證方法是否真的有邊際效益。
- 優點：能快速淘汰看起來漂亮但實際沒用的方法。
- 結論：不是附屬品，是方法選擇的一部分。

## Tier 3: 直接排除

### 7. RL
- 原因：狀態、動作、獎勵都太容易失真，維護成本高。

### 8. HFT
- 原因：硬體、延遲、交易基礎設施都不符合現狀。

### 9. 依賴深度內餡的研究法
- 原因：需要企業內部資料或高度專有資訊，你做不了也不該硬做。

### 10. 重型深度序列模型
- 原因：訓練、調參、驗證成本太高，6G 硬體不划算。

## 最佳主線
- `Change Point / Regime Detection`
- `Cross-sectional + Tree-based`
- `Matrix Profile` 做輔助搜尋
- `Probabilistic / Bayesian` 做風險與狀態補強

## 一句話
- 先切結構，再做橫截面，再用輕量模型驗證。
- 任何需要重硬體、重人力、重內部資訊的方法，直接 drop。
