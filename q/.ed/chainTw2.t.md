# Handoff

## 核心目標
- 先用美國端資料找出與台灣廠商有關聯的公司。
- 再回到台灣市場做交易決策。
- 不先從台灣市場亂掃，不做亂槍打鳥。

## 已收斂的需求
- 入口以美國資料為主。
- 目標不是完整 BOM，而是先找出「有台灣供應商牽扯」的公司。
- 只保留已命中台灣廠商/台灣供應鏈證據的公司。
- 台灣本地資料只做驗證與補證，不當主搜尋入口。

## 資料篩選規則
- 優先條件：
  - 公司集中
  - 資訊集中
  - 可機器化取得
  - 最好能輸出 JSON
  - 免費優先
- 可接受：
  - 需要自建爬蟲，但來源仍集中
  - 資料乾淨不是硬性要求
- 不要：
  - 資料分散在各公司官網
  - 需要大量人工逐站下載
  - 先掃市場再猜命中

## 優先資料來源
### 美國端主來源
- SEC EDGAR
  - 免費
  - 公司集中
  - JSON/API 友善
  - 適合抓 10-K、10-Q、8-K、20-F、6-K
  - 用來找 supplier / customer / supply chain / risk factors
- GDELT
  - 免費新聞事件來源
  - 用來補台灣公司與美國公司之間的事件關聯

### 台灣端輔助來源
- MOPS
  - 台灣公司公告、財報、法說會、重大訊息
  - 用來驗證美國端命中的台灣廠商
- TWSE / TPEx
  - 市場與公司基本資料補強

### 背景/驗證來源
- CBC
- data.gov.tw
- 公司 IR / 年報 / ESG / supplier responsibility report

## 不當主入口的來源
- 各公司自己的官網資料頁
- 分散式新聞網站
- 需要手動翻找的大量 PDF 或簡報
- 以「免費但清洗成本高」為主的零散資料

## 美國公司供應商揭露前提
- 美國公司通常不會頻繁、完整更新所有供應商名單。
- 常見的是依 `materiality` 揭露部分客戶、原料、供應鏈依賴。
- `國籍` 通常不是固定常態揭露欄位。
- 所以不要期待「完整供應商名冊」會常態出現在 10-K 或 8-K。

## 實作流程
1. 從現有資料中抽出公司名稱。
2. 先做台灣命中判定。
3. 只保留已命中台灣廠商或台灣供應鏈證據的公司。
4. 對命中公司再查美國端文件，抽出供應鏈關聯。
5. 用台灣端資料做交叉驗證。
6. 輸出公司名、台灣廠商、證據來源、證據強度、可否進一步交易。

## 命中判定
- 直接命中：
  - 文件明寫台灣公司名、台積電、鴻海、廣達、緯創、和碩、日月光、台達電等
- 間接命中：
  - supplier / customer / manufacturing / assembly / component / module / packaging 等上下游詞彙與台灣實體同時出現
- 弱命中：
  - 只有新聞推測或未證實關聯

## 目前的研究方向
- 主軸是「美國高品質集中資料」反推台灣供應商。
- 台灣資料只用來確認命中是否成立。
- 最終目的不是研究本身，而是回到台灣市場找交易標的。

## 接手時要先記得
- 不要先從台灣市場開始找。
- 不要把免費等同低清洗成本。
- 不要把分散來源放到第一優先。
- 不要先追完整 BOM。
- 先做命中篩選，再做深挖。
