# 自動化新聞管道工具目錄 (2026)

## 管道架構

```
📡 第 1 層：發現  →  🕷️ 第 2 層：採集  →  🧠 第 3 層：提煉  →  🏗️ 第 4 層：發佈
  新聞源 / RSS          抓取 / 反偵測         LLM / JSON          基礎設施 / 監控
```

## 第 3 層 — 🧠 LLM 理解與 JSON 提煉
> 關鍵問題：**如何把原始文字轉成結構化 JSON？** ⚠️ 全部需要 GPU（RTX 4050 6G 最佳化）。

### 本地模型 (4-bit 量化)

| 模型 | 顯存佔用 | 優勢 |
| :--- | :---: | :--- |
| **Qwen 3.6-Coder 3B** | ~2.5 GB | JSON 格式化能力最強 ✅ 首選 |
| **MiniMax M2.7** | < 3 GB | 專為網頁理解設計 |
| **Llama-3.2-3B** | ~2.8 GB | 工具鏈生態最完整 |
| **Phi-4-mini** | ~3.0 GB | 推理邏輯極其精準 |
| **Gemma 4 E4B** | ~3.5 GB | 2026 最新，128K 長文本 |

> 執行建議：量化格式一律用 `GGUF` 或 `EXL2`；`Qwen 3.6-Coder 3B` 佔 2.5GB，剩餘 3.5GB 供系統與瀏覽器。

## 選型決策樹

```
需求是什麼？
│
├── 找新聞源 / RSS / OSINT 監控   →  第 1 層工具
├── 抓取頁面 / 繞過反爬蟲         →  第 2 層工具
├── 理解文字 / 輸出 JSON          →  第 3 層工具  ⚠️ 需 GPU
└── 監控讀者 / 自動發佈 / 建站    →  第 4 層工具
```
